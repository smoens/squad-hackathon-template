# What-If Cards

A deck of 30 stimulus cards for ideation brainstorming. Use them to break patterns, shift perspectives, and spark ideas nobody would have found through linear thinking.

---

## How to Use

**Option A — Deal one per round:**
After each round of brainstorming, deal a random card from the deck. Everyone writes one new idea inspired by the card. (2 min silent write → share.)

**Option B — Spread the table:**
Lay all 30 cards face-up on the table. Participants grab any card that sparks an idea and use it as a prompt. Works well during the cluster/discuss phase.

**Option C — Rescue a stall:**
When brainstorming stalls and sticky notes stop flowing, the facilitator draws a card and reads it aloud: *"OK, new angle..."* Instant redirect.

> **Tip:** Don't use all 30 in one session. 5-8 cards is plenty. Save the rest for day 2 or a different engagement.

## ♠️ Constraints (8 cards)

Cards that force you to design within limits — creativity loves constraints.

| # | Card | Facilitator tip |
|---|------|----------------|
| ♠1 | **What if it had to work offline?** | Great for field workers, factories, remote sites — anywhere connectivity is spotty. |
| ♠2 | **What if the user wears gloves?** | Forces non-touch interfaces: voice, gesture, large physical buttons. Good for industrial. |
| ♠3 | **What if you had zero budget?** | Pushes toward open-source, existing infra, low-code approaches. Surprisingly practical. |
| ♠4 | **What if it had to run on a phone?** | Mobile-first constraint reveals what's truly essential in the solution. |
| ♠5 | **What if you could only use 10 data points?** | Forces prioritization — which signals actually matter? Cuts through big-data hype. |
| ♠6 | **What if it had to work in 3 seconds?** | Latency constraint drives architecture decisions — cached vs. computed, edge vs. cloud. |
| ♠7 | **What if a new employee had to use it on day one?** | Zero-training UX. If it needs a manual, it's too complex. |
| ♠8 | **What if it had to pass a security audit tomorrow?** | Surfaces compliance and privacy considerations you might be hand-waving away. |

## ♥️ Perspectives (8 cards)

Cards that shift the viewpoint — someone else's eyes see different problems.

| # | Card | Facilitator tip |
|---|------|----------------|
| ♥1 | **How would Uber solve this?** | Platform thinking: what if you connected supply and demand differently? |
| ♥2 | **What would a 10-year-old suggest?** | Naively obvious solutions that adults overthink. "Why don't you just...?" |
| ♥3 | **How does nature handle this?** | Biomimicry: swarms, feedback loops, adaptation, resilience patterns. |
| ♥4 | **What would the night shift say?** | Unheard users often have the sharpest insights. Who are you not talking to? |
| ♥5 | **How would a competitor do it?** | Competitive pressure reveals differentiation opportunities. |
| ♥6 | **What would the CFO want to see?** | Financial framing — ROI, cost avoidance, revenue impact. Grounds the idea in business value. |
| ♥7 | **How would this work in a different country?** | Cultural, regulatory, and infrastructure differences reveal assumptions. |
| ♥8 | **What would the customer's customer want?** | One level deeper in the value chain — often reveals the real problem. |

---

## ♦️ Provocations (7 cards)

Cards that flip assumptions — productive discomfort sparks breakthroughs.

| # | Card | Facilitator tip |
|---|------|----------------|
| ♦1 | **What if the opposite was true?** | Inversion thinking. If the problem is "too slow," what if speed was the enemy? |
| ♦2 | **What's the laziest solution?** | Minimum viable effort sometimes reveals the smartest architecture. |
| ♦3 | **What if this problem didn't exist?** | If you magically removed the pain, what would change? Works backward from outcomes. |
| ♦4 | **What if you had to solve it by Friday?** | Urgency as a design constraint. What would you cut? What's actually essential? |
| ♦5 | **What if you made the problem worse on purpose?** | Deliberate sabotage reveals which factors actually drive the issue. |
| ♦6 | **What would you build if you could never update it?** | Forces simplicity and robustness over continuous iteration. |
| ♦7 | **What if the user didn't know they were using it?** | Invisible automation. No dashboards, no buttons — just outcomes that appear. |

---

## ♣️ Technology (7 cards)

Cards that explore tech-enabled possibilities — grounded in what's feasible today.

| # | Card | Facilitator tip |
|---|------|----------------|
| ♣1 | **What if AI could predict this?** | Pattern recognition, forecasting, anomaly detection — where is the signal in the noise? |
| ♣2 | **What if you had real-time data?** | Streaming instead of batch. What decisions change when data is live? |
| ♣3 | **What if voice was the only interface?** | Hands-free scenarios: driving, surgery, factory floor. Forces natural conversation design. |
| ♣4 | **What if you could see it on a digital twin?** | Spatial visualization of physical systems — factories, supply chains, buildings. |
| ♣5 | **What if every device reported its health?** | IoT and telemetry. Predictive maintenance, capacity planning, fleet management. |
| ♣6 | **What if anyone could query the data in plain English?** | Natural language to SQL/KQL. Democratizes data access beyond the analytics team. |
| ♣7 | **What if you could simulate it before you built it?** | Digital rehearsal: scenario modeling, Monte Carlo, what-if analysis on business decisions. |

## Printing Tips

To use as physical cards:
1. Print this page double-sided on card stock
2. Cut along the table rows — each row is one card
3. Color-code the backs by suit (black ♠, red ♥, orange ♦, green ♣) with a marker
4. Shuffle and deal

Or just project this page and point to cards as you use them — works fine too.
